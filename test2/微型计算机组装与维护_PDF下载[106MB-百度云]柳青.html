4．1  安装多操作系统的意义
    随着计算机存储技术的发展，硬盘的容量越来越大，新的微型计算机硬盘配置一般都超
过了40GB，用户不仅可以在硬盘中存储更多的学习、工作、游戏软件，还可以通过安装多个
操作系统来使用更多的应用软件或更好地进行软件开发工作。
    安装多操作系统具有如下意义：
    (1)满足使用不同版本应用软件的需要。有些应用软件有若干版本，分别对应不同的操
作系统版本，一般有For Windows 9X／Me、For Windows 2000／XP、For Windows 2003等版本，
有些还有For Linux版本，而有些应用软件却仅有一种版本。如果计算机中只安装一种操作系
统，可能无法满足不同版本应用软件的需要。
    (2)满足学习非主流或*新操作系统的需要。除Windows 98系统外，微软还不断推出
新版本的操作系统，例如目前流行的Windows 2000、Windows XP和Windows Server 2003等。
可能有些用户还想了解、使用一些其他操作系统，例如Linux等。如果通过安装单操作系统的
方式来使用不同的操作系统，需要重新安装不同的操作系统，如果都从格式化硬盘开始，则太
麻烦了。这时，在一台计算机中安装多操作系统就成了非常好的选择。
    (3)满足系统保护和数据安全的需要。单机安装多操作系统后，可以在某一系统中试用
各种*新版本的应用软件或进行测试，一般情况下不会影响另一操作系统的安全。

    4．2．1  操作系统的引导过程
    系统加电自检通过后，硬盘复位，系统BIOS根据用户指定的启动顺序从软盘、硬盘或光
驱等进行启动。如果设置为硬盘启动系统，BIOS将主引导记录读入内存，然后将控制权交给
主引导程序。接着检查分区表状态，寻找活动分区，*后由主引导程序将控制权交给活动分区
的引导记录，再由引导记录加载操作系统。
    Windows 9X操作系统活动分区引导记录负责读取并执行IO．SYS文件，先初始化一些重
要的系统数据，接着出现启动画面。如果系统中已安装多操作系统，主引导记录将被替换成
该软件的引导代码，这些引导代码允许选择一种操作系统，然后读取并执行该操作系统的基
本代码。
    Windows 2000／XP操作系统通过NTLDR程序装入内存。引导装入程序和多重引导都由一
个具有隐含属性的初始化文件BOOT．iNI控制。该文件包括控制计算机可用的操作系统设置、
默认引导的操作系统和默认等待的时间等信息。
4．2．2硬盘数据的存储与组织   
  FATl6和FAT32文件系统中，硬盘上的数据按照不同的特点和作用大致分为5部分：MBR
区、DBR区、FAT区、DIR区和DATA区。其中，MBR区由分区软件创建，DBR区、FAT
区、DIR区和DATA区由高级格式化程序创建。文件系统写入数据时，只是改写相应的FAT
区、DIR区和DATA区。
    1．MBR区
    MBR(Master Boot Recorder)区即主引导记录区，位于整个硬盘的柱面0、磁头0、扇区
1。在总共512字节的主引导扇区中，MBR的引导程序占用其中的前446个字节，随后的64
个字节为DPT(Disk Partition Table，硬盘分区表)，*后的两个字节“55 AA'’是分区有效结
束标志。由它们共同构成硬盘主引导记录，又称主引导扇区。有时硬盘主引导记录专指MBR
的引导程序。
    2．DBR区
    DBR(DOS Boot Recorder)区即操作系统引导记录区，通常位于硬盘的柱面0、磁头l、
扇区l，是操作系统可以直接访问的**个扇区，包括一个引导程序和一个称为BPB(BIOS
Parameter Block)的本分区参数记录表。
    引导程序的主要任务：当MBR将系统控制权交给它时，判断本分区根目录前两个文件是
不是操作系统的引导文件，如DOS操作系统中的IO．SYS和MSDOS．SYS。低版本的DOS要
求这两个文件必须是前两个文件，即位于根目录的起始处，占用*初的两个目录项。高版本的
DOS已没有这个限制。由于Windows操作系统与DOS是一个家族，因而也沿用这种管理方
式，只是文件名不一样。如果确定文件存在，把IO．SYS读入内存，并把控制权交给IO．SYS。
BPB参数表记录本分区的起始扇区、结束扇区、文件存储格式、硬盘介质描述符、根目录大
小和FAT个数、分配单元的大小等重要参数。
    3．FAT区
   
    在DBR区之后是FAT(File Allocation Table，文件分配表)区。磁盘中，同一个文件的数
据不一定完整地存放在一个连续的区域内，往往会分成若干段，像一条链子一样存放。这种存
储方式称为文件的链式存储。硬盘上的文件常常要进行创建、删除、增长和缩短等操作。这样的
操作做得越多，磁盘上的文件就可能被分得越零碎，每段至少是一簇。由于硬盘上保存着段与段
之间的连接信息即FAT，因此操作系统读取文件时总是能够准确地找到各段的位置并正确读出。
    这种以簇为单位的存储方式也是有缺陷的，主要表现在对空间的利用上。每个文件的*
后一簇都可能有未被完全利用的空间，称为尾簇空间。一般来说，当文件个数比较多时，平均
每个文件要浪费半个簇的空间。
    为实现文件的链式存储，硬盘上必须准确地记录哪些簇已经被文件占用，还必须为每个
已经占用的簇指明存储后继内容的下一个簇的簇号，文件的*后一簇还要指明本簇无后继簇，
这些都由FAT表来保存。FAT表中有很多表项，每项记录一个簇的信息。由于FAT对文件管
理的重要性，因而有一个备份，即在原FAT的后面再建一个同样的FAT。*初形成的FAT中，
所有项都标记为“未占用”。如果磁盘有局部损坏，格式化程序将检测出损坏的簇，在相应的
项中标记为“坏簇”，以后保存文件时不会再使用这个簇。FAT的项数与硬盘上的总簇数相当，
每一项占用的字节数也与总簇数相适应(其中需要存放簇号)。
    FAT的格式有多种，常见的是FATl6和FAT32。其中，FATl6指文件分配表使用两个字
节(即16位)表示一个簇。由于16位分配表*多能管理65536个簇，而每个簇的
存储空间*大只有32KB，因此使用FATl6管理硬盘时，每个分区的*大存储容量只有65536
×32KB=2048MB，即2GB。现在的硬盘容量越来越大，由于FATl6对硬盘分区的容量限制，
当硬盘容量超过2GB之后，只能将硬盘划分成多个2GB的分区才能正常使用。为此，微软公
司从Windows 95 OSR2版本开始使用FAT32标准，即用32位表示一个簇的文件分配表来管理
硬盘文件，这样系统能为文件分配4294967296个簇，在簇同样为32KB时，每个分
区的容量*大可达128TB以上。此外，用FAT32管理硬盘时，每个逻辑盘中簇的大小也比用
FATl6标准管理的同等容量的逻辑盘小很多。由于文件在硬盘上占用的磁盘空间以簇为*小单
位，一个文件即使只有几十个字节，也必须占用整个簇，因此逻辑盘的簇单位容量越小，越能
合理地利用存储空间。所以，FAT32更适用于大容量硬盘。
    4．DIR区
    DIR(Directory)又称文件目录表FDT(File Directory Table)，是根目录区，紧接着第二
FAT表(即备份的FAT表)之后，记录根目录下每个文件(目录)的起始单元、文件的属性
等。定位文件位置时，操作系统根据DIR中的起始单元，结合FAT表确定文件在硬盘中的具
体位置和大小。
    5．DATA区
    DATA(数据)区是真正意义上的数据存储区，位于DIR区之后，占据硬盘上的大部分空间。
    在磁盘中，MBR一般占用63个扇区(实际只占用一个扇区)；DBR占用32个扇区(实

    NTFS是1993年随着Windows NT的**个版本推出而面世的，是一个性能优良的文件
系统。NTFS基于可恢复文件结构而设计，可使用户数据文件不会有丢失或毁坏的危险，适用
于一些要求高安全性，而且在磁盘上存储远远大于FAT文件系统所能处理的巨型文件等场合。
    NTFS文件系统中用“卷”来表示一个逻辑磁盘。卷可以是一个基本分区、一个扩展分区
中的逻辑磁盘，或者是一个被视为非DOS分区的磁盘上的一部分空间。而且，一个卷可以是
被Windows NT指定为一个逻辑驱动器的磁盘空间，它不必是一个磁盘上的相邻空间。
    与FAT相同，NTFS也用“簇”作为*小的分配单位。簇的大小也称为簇因子，由NTFS
格式化程序确定。NTFS支持的簇大小为5 12、1024、2048和4096个字节。这些默认值是一
种在簇太大而造成簇内空间浪费与簇太小而造成产生碎片两者之间的折衷方案。NTFS只与簇
发生关系，与物理扇区的大小无关。
